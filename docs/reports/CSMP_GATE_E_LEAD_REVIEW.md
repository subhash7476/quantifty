## Corroboration checks

- **Construct is the charter-locked 12-1, not a tuned variant.** Score = adjusted return `t−12m → t−1m` (grid indices `i−12` and `i−1`), holding return `t → t+1`; the skipped month (`t−1 → t`) is the standard short-term-reversal guard. Equal-weight, monthly, top-quintile (40) and decile (20) buckets. No parameter is searched; the four locked constants (`FORMATION_LOOKBACK=12`, `SKIP_MONTHS=1`, `N_UNIVERSE=200`, quintile/decile splits) are hard constants.
- **Rebalance grid aligns with gate (c).** The month-end full-session grid (`n_symbols ≥ 200`, MAX per year-month) is 156 sessions, 2010-01-29 → 2022-12-30. All 132 dev-window `universe_membership` rebalance dates fall on grid dates, and no post-2012 grid date is missing from membership — so the `d in members` filter drops nothing silently. The forward-return cutoff correctly excludes the 2022-12-30 formation (its `t+1` would be a 2023 session), yielding 131 scored months.
- **Entity continuity across renames.** Prices are keyed by `entity` (max-turnover symbol per entity-date, the same tie-rule gate (c) used), so a 12-month formation window survives a mid-window ticker change — the `symbol_changes` requirement of Prompt 5 is met structurally, not by a post-hoc patch.
- **Fee model is gate (d), used honestly.** BUY legs pay STT+txn+SEBI+stamp+GST; SELL legs pay STT+txn+SEBI+GST+flat DP; the first rebalance buys the whole book (initial 50% one-way, folded into the disclosed average — conservative); the terminal book is marked, not liquidated (standard for a return series).
- **Inference is honest.** The t-stat is explicitly labelled *naive* (it ignores the 11-month formation overlap); the block bootstrap at L=12 is the load-bearing inference, and its CI lower bound (0.0093) — not the naive t — is what clears rule (A).

## Findings (LOW — documented, non-blocking)

- **F1 (LOW, conservative) — the EW-universe baseline is the formation-complete subset, not the raw 200-name book.** §5's "equal-weight universe" holds only names that have both a complete 12-1 formation window **and** a forward return that month (≈197 of 200), i.e. the same filter the momentum arm passes — not the naive "buy all 200." This is a defensible apples-to-apples choice but is not disclosed in §5. Impact quantified independently: an all-200-members EW book nets **9.07%** vs the delivered **9.16%** — a 9 bp difference that would *widen* the reported spread to ≈6.46%. The direction is conservative and the verdict is unaffected. Recommend a one-line disclosure in §5.
- **F2 (LOW) — the reference-arm cache is written outside criterion-7's two named directories.** `REF_CACHE = data/market_data/ref_nifty200mom30tri.json`. Criterion 7 literally names only `scripts/csmp/` and `docs/reports/`. Mitigation is complete in practice: `data/market_data/` is the program-wide CSMP data location (gates a–c all write there) and is `.gitignore`d, so there is **no tracked diff**, and the report footer discloses the path explicitly. Noted for literal-criterion awareness, not as a fence breach.
- **F3 (LOW, cosmetic) — the printed sealed-window assertion covers only the adjusted view.** All three input queries are bounded by `≤ DEV_END`, but only the `equity_bhavcopy_adjusted` boundary is echoed (`MAX = 2022-12-30 (OK)`); the membership and calendar `MAX` are bounded-but-not-displayed. The fence is airtight (I confirmed the loaded price map tops out at 2022-12-30), so this is a display gap, not a leak. Prompt 5's "printed in the audit" is satisfied for the price view that matters; echoing the other two would fully honor the letter of it.
- **F4 (LOW) — code comment overstates a restriction the SQL does not apply.** The price-load comment says it is "Restricted to entities that are ever a dev-window member," but the query joins every `universe_eligibility` symbol (4,132 symbols / 3,620 entities) with no membership filter. Correctness is unaffected — non-member rows are simply never looked up — but the comment should match the code (or the filter should be added to trim the load).
- **F5 (INFORMATIONAL) — the 40/40 and 20/20 buckets assume ≥ 80 / ≥ 40 names present.** Guaranteed here (dev-window membership is **exactly 200** every month, ≈197 after exclusions), so top and bottom buckets never overlap. The invariant is inherited from gate (c), not enforced locally; a note only, since gate (c)'s "exactly 200" makes it structurally safe for this window.
