## Task 2 — Contract v2: make Sharpe the declared quantity for PnL metrics

### The structural insight to implement

For a per-trade PnL metric the noncentrality parameter reduces to a function of Sharpe and elapsed
time alone:

```
ncp = (delta/sd)·√n = (S/√c)·√(c·T) = S·√T
```

where `S` = annualized Sharpe, `c` = formations/year, `T` = years. **SD cancels.** Declaring
separate `delta` and `sd` bands for `per_trade_pnl` is therefore not merely risky — it is
*over-parameterised*: two numbers are declared where the model admits one, and the surplus degree
of freedom is exactly what the crossed corner exploited.

### Changes to `governance/rfa/declaration.py`

Add fields: `sharpe_lo: float | None`, `sharpe_hi: float | None`, `cadence_per_year: int | None`,
`sharpe_provenance: str = ""`. Keep `delta_*`/`sd_*` (still used by `rank_ic`).

Extend `validate(decl)` to branch on `decl.metric`:

- **`metric == "per_trade_pnl"`** — require `sharpe_lo`, `sharpe_hi`, `cadence_per_year`, and
  non-empty `sharpe_provenance`. Require `0 < sharpe_lo <= sharpe_hi` and `cadence_per_year >= 1`.
  **Raise** if `delta_lo`/`delta_hi`/`sd_lo`/`sd_hi` are also supplied — a PnL declaration must not
  carry the redundant parameterisation that caused this defect. Error message must name the reason,
  e.g. `"per_trade_pnl declares a Sharpe band; delta/sd are derived. Supplying both re-introduces the coupled-band defect (RFA_GATE_O1_REVIEW.md §1)."`
- **`metric == "rank_ic"`** — current behaviour unchanged (delta/sd bands, independence defensible
  because IC dispersion is largely a sample-size artifact). **Raise** if a Sharpe band is supplied.

### Changes to `scripts/rfa/gate.py`

Branch the corner construction on metric:

- **`per_trade_pnl`**: per-formation Sharpe `s_f = sharpe_hi / √cadence_per_year`. Optimistic corner
  power is `power_at(s_f, 1.0, n_available, two_sided)`. Central uses `(sharpe_lo+sharpe_hi)/2`,
  pessimistic uses `sharpe_lo`. **There is no crossed corner** — the failure mode is structurally
  unreachable, which is the point.
- **`rank_ic`**: unchanged.

Extend `Verdict` so the report can distinguish the two (e.g. `corner_sharpe: float | None`). Keep
existing field names intact where they still apply.

### `METHODOLOGY_VERSION` → `"2.0.0"`

This is a breaking contract change. `evaluate()` already hard-fails on version mismatch — so
`o1_vrp` (pinned `1.0.0`, unedited) will now **raise** if anyone runs it. That is the intended
enforcement: a withdrawn declaration should not be re-runnable. Add a test asserting exactly this.
