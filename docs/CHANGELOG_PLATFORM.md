## 2026-07-02 ÔÇö MM12.3 ÔÇö GuardedSignalSource (runtime boundary guard)

Implemented `core/runtime/guarded_signal_source.py`: the platform-owned `SignalSource` decorator that enforces the certified Strategy Contract at runtime (ADR-018 reject-and-journal boundary validation; ADR-019 quarantine-and-continue fault policy ÔÇö combined in one component per the Technical Lead's non-blocking note). Per bar: short-circuits `[]` once quarantined; runs the inner source inside a fault boundary; validates return shape, timestamp discipline, and mandatory entry risk metadata (`sl_distance`/`risk_r` > 0 on BUY/SELL); drops contract violators and journals `SIGNAL_CONTRACT_REJECTED` while clean siblings still route; quarantines on an uncaught fault or malformed return (journals `STRATEGY_ERROR` + `STRATEGY_QUARANTINED`, edge-triggered, alerts critical, terminal ÔÇö no retry, no auto-flatten). `on_start` faults are a startup refusal (journal, re-raise); `on_stop` faults are logged and swallowed. All of the guard's own side channels (journal/telemetry/alerter) are individually fault-contained so a broken sink can never let an exception escape into `LoopDriver`. Added 3 additive `EventType` members and 3 additive `RuntimeMetric` counters (non-breaking). 35 new fault-injection tests cover every failure mode in the ADR-019 fault matrix, including the guard passing the MM12.2 conformance suite itself (both with an inert and a permanently-raising inner source). **Zero diffs** in `core/runtime/driver.py`, `core/execution/handler.py`, `core/runtime/signal_source.py`, `core/runtime/conformance.py` ÔÇö the MM12.3 acceptance criterion.
