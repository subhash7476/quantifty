## Strategy Logic

### Session Structure (all times IST, UTC+5:30)

**Session 1 — London sweep of Asian range**
- Pre-session range: 9:30 AM – 1:30 PM IST (late Asian session — tight range, fewer false sweeps)
- Trading window: 1:30 PM – 7:00 PM IST (London open + 5.5 hours)

**Session 2 — NY sweep of London range**
- Pre-session range: 1:30 PM – 7:00 PM IST (London range)
- Trading window: 7:00 PM – 11:00 PM IST (NY open + 4 hours)

### Entry Sequence (per session)

1. **Sweep detection**: Price breaks pre-session high or low by ≥ 0.25 × M15 ATR
2. **Sweep filter** *(optional)*: K-line classifier scores the sweep; skips if prob < threshold
3. **Structure shift**: After sweep, detect lower high (bearish) or higher low (bullish) + displacement candle ≥ 1.2 × M5 ATR
4. **Pullback entry**: Price retraces into displacement candle body
5. **Risk calc**: SL = sweep extreme + 0.15 × M15 ATR buffer; TP = entry_fill ± 2R (computed from **actual fill price**, not structural zone)

> **Note on TP**: `entry_price` in the setup is the structural zone level. The live trader uses the actual MT5 fill price to compute TP, ensuring a true 2R regardless of slippage direction. SL always stays at the structural sweep level.

### Exit Rules

| Condition | Action |
|-----------|--------|
| Price hits take profit | Close at 2R |
| Price hits stop loss | Close at -1R |
| Session cutoff reached | Close at market (TIME_CUTOFF) |
| FTMO daily limit hit | Stop all trading for the day |
| FTMO overall limit hit | Stop all trading permanently |

## Key Design Decisions

### Partial Exit at 1R — Backtest Results (2026-03-13)

| Config | Trades | WR | PF | Total P&L | Max Loss Streak | Exit Breakdown |
|--------|--------|----|----|-----------|-----------------|----------------|
| Baseline | 348 | 41.1% | 1.12 | $10,598 | 9 | TP 68 / SL 147 / TC 133 |
| + Partial exit | 348 | 51.4% | 1.12 | $8,137 | 6 | TP 56 / SL 119 / BE 58 / TC 115 |
| + Sweep filter | 196 | 57.1% | 1.93 | $24,576 | 4 | TP 49 / SL 53 / TC 94 |
| + Both | 196 | 64.3% | **2.0** | $18,833 | **3** | TP 39 / SL 43 / BE 32 / TC 82 |

**Verdict**: Partial exit improves WR, PF, and max loss streak but reduces absolute P&L (~$5.7K lower with filter). The right choice depends on the goal:
- **Maximum P&L**: No partial exit (sweep filter only → $24,576)
- **Lowest drawdown / smoothest equity**: Partial exit + filter (WR 64%, loss streak 3, PF 2.0)

**For FTMO Phase 1** (10% profit target, 5% daily limit): no partial is better — you pass faster with higher P&L. After passing, partial exit is preferable for the funded phase where capital preservation matters more.

| **XAGUSD (added)** | Same ICT sweep thesis as Gold; higher ATR volatility; 5× point value means smaller lot sizes; needs separate backtest on XAGUSD M5 data |

### Why 9:30 AM pre-session start (not 5:30 AM)

- 8-hour Asian range (5:30 AM – 1:30 PM) = $35 average range → too wide → false sweeps
- 4-hour range (9:30 AM – 1:30 PM) = $25 average range → tighter → win rate 28% → 35%
- Narrower range = more decisive sweeps = cleaner reversal setups

### Why dual sessions

- Single London session: 181 trades, +$5,973 (not enough for Phase 1 in time)
- Dual sessions: 348 trades, +$10,598 (Phase 1 passed Jan 2026)
- NY session uses London range as reference — different structural level, same ICT logic

### Critical bugs fixed during development

1. **`new_day()` did not reset `consecutive_losses`** → first 2 losses in all of history blocked ALL future trading → backtest showed only 2 trades
2. **Session classification via `apply()` on 99K rows × 367 dates** → hanging (replaced with vectorised `_classify_all()`)
3. **`get_ny_session_bars()` called per-day on full DataFrame** → replaced with `get_all_ny_bars()` pre-grouping
4. **`NY_END = 19:00` in `detector.py` and `risk.py` blocked all Session 2 trades** → propagated `cutoff` parameter through all detection functions
5. **Engine defaulted to $50K starting balance** → fixed to use `ACCOUNT_SIZE` from config
6. **Session 2 `scan_session()` called without cutoff parameter** → all S2 setups returned `None` in structure shift detection

## K-line Sweep Quality Filter (Mar 2026)

Optional ML layer to classify whether a detected sweep is a genuine ICT reversal or a continuation trap, before structure shift detection runs.

### How it works

1. **Tokenizer** (`ftmo/kline_tokenizer.py`): encodes each M5 candle into a discrete token (0–63) using K-means clustering on 3 normalized features — body ratio, upper wick/ATR, lower wick/ATR. Vocabulary trained once on all 100K historical bars.

2. **Classifier** (`ftmo/sweep_classifier.py`): at sweep detection time, takes the 20-bar pre-sweep sequence → 64-dim token histogram + 3 structural features (sweep extension/ATR, sweep-bar body ratio, close retrace within sweep bar) → LogisticRegression → reversal probability.

3. **Gate**: if probability < threshold (0.43), sweep is skipped entirely. No structure shift detection, no setup. Threshold is F1-optimized on training data.

### Per-Session Models (S1 / S2 Split)

Session 1 (London sweep of Asian range) and Session 2 (NY sweep of London range) have different structural dynamics. Training separate models captures session-specific patterns.

| Model | Session | File |
|-------|---------|------|
| Combined | Both | `ftmo/sweep_clf.pkl` |
| Session 1 | London (13:30–19:00 IST) | `ftmo/sweep_clf_s1.pkl` |
| Session 2 | NY (19:00–23:00 IST) | `ftmo/sweep_clf_s2.pkl` |

**Fallback chain**: per-session model → combined model → no filter (each step only if the previous file is missing).

### Training Pipeline

```bash
# Step 1: extract labeled sweeps with session tag (4-tuple format)
python scripts/label_sweeps.py
# Output: sweep_labels.pkl — list of (hist, struct_feats, label, session_num)

python scripts/train_sweep_clf.py
```

### Integration

- **Backtest**: `FTMOBacktestEngine(df, sweep_classifier=clf_s1, sweep_classifier_s2=clf_s2)` — S1 and S2 use their respective models. CLI `--sweep-filter` auto-loads per-session if available.
- **Live**: `MT5LiveTrader` loads `_sweep_clf_s1` and `_sweep_clf_s2` on startup; routes S1 classifier to the London scan and S2 to the NY scan.

### Limitations

- CV ROC-AUC = 0.523 on combined model (modest signal; logistic preferred at ~350 samples)
- Per-session models trained on ~175 samples each — expect similar AUC until more live data accumulates
- All thresholds are in-sample optimized — validate on held-out period before raising risk
- Retrain monthly as `trade_log_{sym}.csv` accumulates new live samples
