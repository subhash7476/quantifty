   > **Implementation note — single source of truth for `KILL_SWITCH_ACTIVATED`.** The watchdog phase (E) emits `KILL_SWITCH_ACTIVATED` via a **data-health proxy** (the `watchdog.data_healthy` True→False edge) because the watchdog trip was the sole kill-switch cause while the loop was execution-free. **Execution routing (G) has now landed (routing sub-slice), so the handler's kill switch can flip from other causes (drawdown, broker, daily-limit) — but the driver still observes only `watchdog.data_healthy`, so a handler-caused trip is currently un-journaled (a missed emission, not a double-emit).** The Phase G consolidation increment **must migrate to `ExecutionHandler` kill-switch ownership** — emit `KILL_SWITCH_ACTIVATED` from a single observation of the handler's kill-switch edge (`_kill_switched`, §10.7) — and **remove the data-health proxy emission** from the watchdog path, so each kill-switch activation is journaled exactly once regardless of cause.
